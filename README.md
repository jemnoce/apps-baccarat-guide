# apps-baccarat-guide

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态 HTML 文件的形式存在，不依赖于特定的后端服务或数据库。仓库主要作为页面资料的集合与分发点，便于管理和访问。

## 项目简介

这是一个面向 HTML 页面的归档仓库，内容以独立、自包含的网页文件为主。每个页面均可以直接在浏览器中打开，无需额外配置。仓库本身不针对任何单一域名或特定网站，仅为页面文件的存储和索引提供统一入口。

## 目录说明

```
.
├── index.html          # 仓库入口页面，提供页面导航或说明
└── pages/              # 存放各个独立 HTML 页面的目录
```

- `index.html`：仓库首页，通常包含指向其他页面的链接或简要介绍。
- `pages/`：所有归档的 HTML 页面均放置在此目录下，可按需组织子目录。

## 页面归档说明

- 所有页面均为静态 HTML，支持现代浏览器直接运行。
- 页面内容不包含外部依赖（如 CDN 资源）或仅引用通用库。
- 文件命名建议使用英文小写字母、数字及连字符，避免特殊字符。
- 每个页面应保持独立，不依赖同目录下其他文件（除非明确说明）。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，并在 `index.html` 中添加相应链接。
- 更新页面：直接替换对应文件，保持文件名不变。
- 删除页面：移除文件并更新 `index.html` 中的链接。
- 建议在提交时附带简要说明（commit message），描述新增或修改的内容。

## 使用方式

1. 克隆本仓库到本地：
   ```bash
   git clone https://github.com/your-username/apps-baccarat-guide.git
   ```
2. 直接在浏览器中打开 `index.html` 或 `pages/` 下的任意页面。
3. 也可通过 GitHub Pages 等静态托管服务发布（需自行配置）。

## 注意事项

- 本仓库仅作为页面归档使用，不提供任何形式的服务或数据处理功能。
- 所有页面内容由仓库维护者自行负责，请确保符合所在地区的法律法规。
- 不鼓励在仓库中添加任何形式的营销、推广或诱导点击内容。
